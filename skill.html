.claude/skills/dev-lifecycle/SKILL.md

--- name: dev-lifecycle description: | 7단계 개발 라이프사이클을 자동으로 실행. 생각→계획→구축→검토→테스트→출시→성찰 allowed-tools: - Bash, Read, Write, Edit, Agent... --- # Phase 1: 생각 (Think) 사용자 요구사항을 분석... # Phase 2: 계획 (Plan) 아키텍처와 파일 구조 설계... # Phase 3: 구축 (Build) 시멘틱 HTML, CSS 변수, 접근성... # Phase 4: 검토 (Review) 접근성, 에러핸들링, DRY 체크... # Phase 5: 테스트 (Test) 기능 테스트, 엣지케이스, 반응형... # Phase 6: 출시 (Ship) # Phase 7: 성찰 (Reflect)

이 파일 하나가 전체 개발 프로세스를 정의합니다.

개발 과정 ② 스킬: 요구사항 인터뷰

코드 작성 전, AI가 6가지 핵심 질문을 던진다

AI가 질문

Q1 "핵심 문제는 무엇인가요?"
Q2 "사용할 사람은 누구?"
Q3 "기능 우선순위를 정해주세요"
Q4-6 기술 제약 · 디자인 · 성공 기준

사용자가 선택

→ "할 일 관리 + 진행률 시각화"
→ B) 팀 내부 사용
→ 필수: CRUD, 검색, 필터
→ 선택: 드래그, 진행률
→ B) 모던 프리미엄 디자인

→ 코드 한 줄 쓰기 전에, AI와 사용자가 함께 방향을 정한다

개발 과정 ③ 구축 → 자동 검토 → 자동 수정

인터뷰 결과를 기반으로 자동 실행

1
요구사항 기반 코드 생성 — Q1-Q6에서 정한 방향 반영/build
2
/review 자동 실행 → 6개 이슈 발견 → 자동 수정
접근성 누락, 에러핸들링 없음, ID충돌, 코드중복, 이중호출, 전역변수
3
/qa 자동 실행 → 1개 버그 발견 → 자동 수정
emptyMsg 참조 오류 (this 바인딩 누락)
4
최종 결과물 — 검색, 진행률, 일괄작업 + 다크 글래스모피즘 UI
인터뷰에서 "모던 프리미엄" 선택 → 디자인 자동 반영

결과 비교: 같은 시작, 다른 결과

스킬 없이 만든 버전

  • ✓ 기본 CRUD + 필터
  • ✗ 검색 없음
  • ✗ 진행률 없음
  • ✗ 접근성 없음
  • ✗ 에러 핸들링 없음
VS

스킬로 만든 버전

  • ✓ CRUD + 검색 + 진행률
  • ✓ 필터 카운트 표시
  • ✓ 일괄삭제 / 전체토글
  • ✓ 접근성 완비
  • ✓ 다크 모던 UI

/review + /qa가 발견한 것들

  • 리뷰 localStorage 파싱 시 에러 핸들링 누락 → try-catch 추가
  • 리뷰 Date.now() ID 충돌 가능 → 랜덤 접미사 추가
  • 리뷰 접근성 속성 누락 → aria-label, aria-live, aria-pressed
  • 리뷰 드래그 이벤트 코드 중복 → 헬퍼 메서드 추출
  • 리뷰 편집 시 blur/Enter 이중 호출 → finished 플래그
  • 리뷰 전역 변수 사용 → TodoApp 클래스로 캡슐화
  • QA emptyMsg 참조 오류 → this.emptyMsg로 수정

7개 이슈 자동 발견 → 전부 자동 수정
